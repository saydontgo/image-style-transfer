# 风格迁移实验报告模板

## 1. 题目

基于深度学习的图像风格迁移实验

## 2. 算法理论介绍

本实验使用的是 `Johnson` 提出的快速风格迁移方法。先说我自己的理解：它不是像传统方法那样，拿一张图片反复优化几百轮，慢慢“磨”出风格图；它更像是先训练一个专门会画某种风格的网络，训练好以后，只要把内容图丢进去，它就能很快输出一张风格化结果。所以这个方法最大的特点就是：`训练比较花时间，但推理非常快`。

这个项目里的整体思路，可以比较口语化地理解成下面几步：

- 第一步，准备一堆普通内容图，再准备一张目标风格图，比如 `mosaic` 或 `candy`。
- 第二步，训练一个前馈卷积网络，让它学会把“普通照片”直接变成“带目标画风的照片”。
- 第三步，训练时不直接拿输出图和风格图做像素级硬对硬比较，因为那样其实不太合理。风格迁移更关心“像不像这种画风”，而不是“每个像素是不是完全一样”。
- 第四步，所以这里会借助一个预训练好的 `VGG16` 网络来提取特征，再用这些特征来计算损失。可以把 `VGG16` 理解成一个已经见过很多图片的“老裁判”，它不负责生成图，只负责判断生成结果在内容和风格上像不像。

从代码实现上看，这个项目的生成网络是 `TransformerNet`。它先做下采样，把图像压缩到更适合提特征的空间；中间用多个残差块学习“这张图应该往什么风格上改”；最后再上采样，把结果恢复成图片大小。除此之外，项目还额外加了跳连、门控和细节分支，这样做的直观目的就是：在风格明显的同时，尽量把边缘、文字轮廓和细小结构保住，不要一风格化就全糊掉。

训练时主要优化下面几类损失：

- `内容损失（Content Loss）`：这个损失控制“图里原本是什么东西”不要变得太离谱。比如人还是人，房子还是房子，物体的大致位置和结构尽量保留。代码里不是只看最浅层特征，而是看了 `relu2_2` 和 `relu3_3` 这些中间层，所以它关注的更像是结构和语义，而不是单纯的像素值。
- `风格损失（Style Loss）`：这个损失负责让输出图“看起来像目标画风”。这里用到的是 `Gram Matrix`。我理解它时，会把它想成一种“统计特征之间关系”的方法。它不在乎这根笔触具体出现在左上角还是右下角，而更在乎整张图是不是有类似的颜色关系、纹理感和笔触分布。所以它特别适合描述“风格”这种比较抽象的东西。
- `总变分损失（TV Loss）`：这个损失主要是让图像更平滑一点，减少噪声、棋盘格和一些不自然的小毛刺。可以把它理解成一个“别太躁，画面收一收”的约束。
- `边缘保持损失（Edge Loss）`：这是这个项目里比较贴近实际效果优化的一点。代码会先从亮度信息里提边缘，再约束生成图不要把原图的重要边缘破坏得太严重。这样做的好处是，像建筑轮廓、文字边缘、人物五官这些位置，通常能保留得更稳一点。

整个训练目标可以概括成：

```text
L_total = alpha * L_content + beta * L_style + gamma * L_tv + delta * L_edge
```

其中：

- `alpha` 是内容损失权重
- `beta` 是风格损失权重
- `gamma` 是总变分损失权重
- `delta` 是边缘保持损失权重

这些权重本质上就是在回答一个问题：`到底是更像原图，还是更像画风？`  
如果 `style weight` 很大，风格会更强，颜色和纹理更明显，但原图结构可能被盖过去；如果 `content weight` 更大，原图会更稳，但风格味道可能没那么浓。`tv weight` 和 `edge weight` 则更像是在帮结果“收边”和“保细节”。

还有一个这个方法很经典、也很现实的特点，就是 `一种风格通常对应一个模型`。也就是说，如果想做 `mosaic` 和 `candy` 两种风格，一般就训练两个模型，而不是一个模型全包。这个项目的训练脚本也是按这个思路设计的：每次训练都会保存配置、阶段 checkpoint 和最终权重，再配合归档脚本做版本整理。这样做虽然不直接属于“算法公式”的部分，但非常符合这个方法的使用方式，因为不同风格本来就应该分开训练、分开管理。

总的来说，这个算法的核心不是“硬把两张图拼起来”，而是训练一个网络，让它学会在`保留内容结构`的前提下，尽量把图像改造成目标艺术风格。它比早期那种逐张优化的方法快很多，也更适合做课程实验和实际演示。

## 3. 代码介绍

本实验代码由以下模块组成：

- `train.py`
  - 负责训练单风格模型
  - 输入为内容数据集与一张风格图
- `stylize.py`
  - 负责使用训练好的模型对图片进行风格迁移
- `compare_models.py`
  - 负责将公开预训练模型与自训练模型在相同内容图上的效果进行可视化对比
- `style_transfer/models/transformer_net.py`
  - 前馈风格迁移网络
- `style_transfer/models/loss_network.py`
  - VGG 感知损失网络与 Gram Matrix 计算

可说明的实现要点：

- 使用 `InstanceNorm2d` 提升风格迁移稳定性
- 使用 `AMP mixed precision` 降低显存占用
- 使用单风格单模型设计，便于完成“两种风格”实验

## 4. 实验设置

### 4.1 硬件环境

- GPU：NVIDIA 5080 16GB
- 系统环境：Windows 11 + WSL2
- 深度学习框架：PyTorch

### 4.2 数据集

- 内容数据集：MS-COCO
- 风格图 1：`mosaic.jpg`
- 风格图 2：`candy.jpg`

可填写示例：

```text
image_size = 256
style_size = 512
batch_size = 8
epochs = 2
subset_size = 20000
learning_rate = 2e-4
style_weight = 1e5
content_weight = 1.0
tv_weight = 1e-6
```

### 4.4 对比方案

- 方案 A：下载的公开预训练模型
- 方案 B：自己训练的模型
- 对比原则：
  - 使用同一批测试内容图
  - 尽量使用相同架构
  - 尽量使用相同内容训练集

可以讨论：

- `style_weight` 增大后，风格更明显，但原图结构可能减弱
- `content_weight` 增大后，内容更清晰，但风格可能偏弱
- `tv_weight` 过大时图像会变平滑甚至发糊
- `subset_size` 增大后结果更稳定，但训练时间更长

## 6. 自己的见解和讨论

可参考以下思路展开：

- 快速风格迁移比 Gatys 优化式方法推理更快，更适合实际应用
- 但快速方法通常需要“一种风格训练一个模型”，灵活性不如优化式方法
- 单卡训练时，合理控制图像分辨率和 batch size 很重要
- 自训练模型如果训练轮数和数据量不足，通常会弱于公开成熟模型
- 若后续继续优化，可以考虑：
  - 增加训练集规模
  - 延长训练轮数
  - 更细致地调节 style/content loss 权重
  - 尝试多风格统一模型

## 7. 结论

- 完成了至少两种艺术风格实验
- 完成了公开模型与自训练模型对比
- 验证了感知损失方法在图像风格迁移任务中的有效性
